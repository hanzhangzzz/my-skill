# X/Twitter 推文结构模式分析

## 模式 1：推文 + 引用 Article（小小东模式）

**特征**：
- 推文正文包含"佐料"（用户补充的要求）
- 引用 Twitter Article 包含"汤底"（完整提示词模板）
- Article 需要登录才能查看

**示例**：
```
推文正文：
GPT2 x 乐高字体 x 无限快乐 x 提示词
这个提示词佐料：
【主题】 
文字：快乐无价 
比例1:2 
适合做成手机壁纸

引用文章：
GPT2实战： 乐高 x 文字 x 无限可能
（完整提示词在文章中）
```

**提取策略**：
1. 用浏览器提取推文正文（包含佐料）
2. 提取代码块内容（如果有）
3. 记录文章标题和摘要
4. 完整提示词需要登录后打开文章

## 模式 2：推文 + 代码块

**特征**：
- 推文中直接包含代码块（` ``` `）
- 代码块中是完整提示词

**示例**：
```
思来想去， GPT-image2 最牛的应用情境：
算命
你先上传手掌正面照，然后用如下提示词 Prompt ⤵️

```
给你这个掌纹。请通过中国的玄学、指纹分析...
```
```

**提取策略**：
1. 用浏览器提取完整推文文本
2. 用正则 `re.findall(r'```(.*?)```', text, re.DOTALL)` 提取代码块
3. 代码块内容就是完整提示词

## 模式 3：推文 + t.co 链接（无代码块）

**特征**：
- 推文包含 t.co 链接
- 没有代码块
- 链接指向文章或其他内容

**提取策略**：
1. 用浏览器提取推文正文
2. 记录 t.co 链接
3. 尝试解析链接（可能需要登录）

## 模式 4：纯文字推文

**特征**：
- 没有代码块
- 没有 t.co 链接
- 直接包含提示词或讨论

**提取策略**：
1. 直接从 xreach text 字段提取
2. 用正则提取引号中的内容

## 关键发现

### xreach text 字段截断问题

**问题**：xreach 返回的 `text` 字段会被截断，特别是：
- 包含代码块的推文（代码块内容会变成 `...`）
- 超过 280 字符的长推文
- 引用 Article 的推文

**解决**：必须用浏览器提取完整内容

### Twitter Article 登录限制

**问题**：`x.com/i/article/xxx` 需要登录才能查看完整内容

**影响**：
- 无法直接获取文章中的完整提示词
- 只能看到文章标题和摘要

**解决**：
1. 登录后用浏览器打开文章
2. 或者从推文正文中提取"佐料"部分
3. 联系作者获取完整提示词库

### Jina Reader 提取 Article 内容（重要新发现）

**`r.jina.ai` 对 Article 格式推文有效**，但对 Quote Tweet 无效：

| 推文类型 | r.jina.ai 能读到 | 能读到的内容 |
|----------|------------------|-------------|
| **Article 格式**（`x.com/xiaoxiaodong01/status/{id}`） | ✅ 有效 | 完整正文 + 图片 alt text（提示词常在这里） |
| **Quote Tweet**（`isQuote=true`，引用其他推文） | ❌ 只读到"Quote"标题 | 引用内容被 X 的登录墙挡住 |

**验证方法**：
```bash
curl -s "https://r.jina.ai/https://x.com/xiaoxiaodong01/status/{id}" -H "Accept: text/markdown"
```

**从 Article 提取图片 alt text（提示词来源）**：
```python
import re
all_alts = re.findall(r'Image (\d+): ([^\]]{10,})', content)
for idx, alt in all_alts:
    print(f"Image {idx}: {alt}")
```

**实战案例**：
- 推文 `2051854592732991621`（情头）是 Article 格式 → ✅ Jina 读到图片 alt text "简单来说，先垫图你喜欢的图片..."
- 推文 `2051898673395822760`（小东东技巧 x 贰）isQuote=true → ❌ Jina 只能读到 Quote 标题

**新工作流（针对 Article 格式推文）**：
1. 优先用 `r.jina.ai` 提取 Article 推文内容 + 图片 alt text
2. 检查图片 alt text 是否包含提示词关键词（`提示词`、`任务`、`结构要求`、`风格`等）
3. 如果 alt text 是完整的提示词 → 直接使用
4. 如果 alt text 只有摘要或缺失 → 降级到浏览器提取
5. Quote Tweet → 必须用浏览器（且可能需要登录）

## 模式 5：推文 + 引用 Article + 佐料（小小东典型模式）

**特征**：
- 推文正文：主题 + 佐料（用户自定义要求）
- 引用 Article：完整提示词"汤底"
- xreach 返回的 text 字段只包含推文正文，不包含 Article 内容

**示例**：
```
推文正文：
GPT2 x 乐高字体 x 无限快乐 x 提示词
这个提示词佐料：
【主题】 
文字：快乐无价 
比例1:2 
适合做成手机壁纸

引用文章：
GPT2实战： 乐高 x 文字 x 无限可能
（完整提示词在文章中，需登录）
```

**提取策略**：
1. xreach 获取推文正文 → 提取"佐料"部分
2. 浏览器提取完整推文文本 → 获取文章标题和摘要
3. 文章完整内容需要登录 → 记录链接，标记为"需登录"

**关键教训**：
- 用户期望能自动获取所有内容，不要求手动登录
- 但 Article 确实需要登录，这是 X 平台的限制
- 应该在开始时就说明这个限制，而不是最后才说
